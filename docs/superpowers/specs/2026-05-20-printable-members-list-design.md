# Printable Members List (New Tab)

**Date:** 2026-05-20
**Files affected:**
`routes/web.php` (new route),
`app/Http/Controllers/PortalController.php` (new `printable()` method),
`resources/views/portal/members/print.blade.php` (new view),
`resources/views/portal/members/index.blade.php` (Print button).

## Goal

The Print button on the staff portal members page should open a clean,
printable report in a **new browser tab**, showing the **same filtered data**
currently displayed — all matching members, not just the visible page. The
user prints that tab with the browser (Ctrl+P).

## Current behaviour

The members page already has a Print button, but it runs
`onclick="window.print()"` — which prints the live page including the sidebar,
filters card and pagination. There is also a PDF export (`exportPdf` →
downloads a file) and a CSV export. None opens a clean printable HTML report
in a tab.

## Changes

### 1. Route

Add inside the protected portal group in `routes/web.php`, after the existing
`members.export.pdf` route, with the same `auth` + `active.user` middleware:

```php
Route::get('/members/print', [PortalController::class, 'printable'])
    ->name('members.print');
```

### 2. Controller — `printable(Request $request, WildApricotService $wa)`

- Reads the same `$filters` as `members()` / `exportPdf()` and applies the
  **identical authorisation enforcement**: zone-level users get their zone
  forced; center-level users get zone + center forced. This both makes the
  printout match the members page and blocks URL tampering.
- Fetches **all** filtered members with the batch loop already used by
  `exportCsv` — `getMembersPage($page, 500, $filters)` repeated until a partial
  page is returned — so the report shows the true filtered total, not 25.
- Builds the same `$filterLabels` map `exportPdf` builds (human-readable
  applied-filter labels).
- Returns `view('portal.members.print', [...])` with: members array, total
  count, the current date, the user's role label, and `$filterLabels`.

### 3. View — `portal/members/print.blade.php`

A standalone HTML page, browser-print sized (not the tiny PDF fonts of
`pdf.blade.php`):

- Centred green title **"ISGH Admin Staff"**, subtitle
  **"Members List - <date>"**, line **"Total Members: N"**.
- 8-column table: `#`, Member Name, Membership Type, Zone / Masjid, Complete
  Address, ZIP, Dates (Joined / Renewal), Status — same data shape and
  status-badge logic as `pdf.blade.php`.
- Footer: **"Generated by ISGH Admin Staff Dashboard"**.
- A small "Print this page" button at the top calling `window.print()`,
  hidden under `@media print`.
- A `@media print` block for clean output.

### 4. Print button — `index.blade.php`

Replace the `<button onclick="window.print()">` (~line 191) with an `<a>` that
opens the new route in a new tab, carrying the current filters — same pattern
as the existing CSV/PDF export links:

```blade
<a href="{{ route('portal.members.print', request()->query()) }}"
   target="_blank" rel="noopener" class="btn btn-secondary btn-sm" ...>
```

Same icon and "Print" label; a link instead of a button.

## Edge cases

- **Filters carried** via `request()->query()` — the printout reflects the
  active Zone / Masjid / ZIP / status / search.
- **Authorisation** re-enforced server-side in `printable()` — a zone/center
  user cannot print outside their scope by editing the URL.
- **Empty result** → the table shows a "No members found" row.

## Testing

1. `php artisan view:clear`.
2. On the members page, apply a filter (e.g. a Zone), click Print → a new tab
   opens the "ISGH Admin Staff" report listing all members for that filter
   with the correct "Total Members" count.
3. Ctrl+P in that tab → clean printout, no app chrome.
4. As a center-level user, confirm the printout is scoped to their center.

## Out of scope

- Changing the existing PDF or CSV exports.
- The member portal (member-facing) pages.
